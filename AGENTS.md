# Agent Rules

1. Preserve the immutable checks-and-balances requirement.
2. Prefer deterministic rules and classical ML before semantic agents.
3. Never merge observed content directly into belief or adaptive training.
4. Never let an immune agent modify canonical state without an explicit promotion path.
5. Treat the Protomentat as the final authority for consequential ambiguity.
6. Keep the first implementation one process and one SQLite database.
7. Add infrastructure only after measured failure of the simpler path.
8. Every capability must declare inputs, outputs, authority, provenance, failure behavior, and rollback.
