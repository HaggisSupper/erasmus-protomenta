# Agentic Expansion Program

This document defines the dependency order for expanding the Erasmus–Protomentat kernel through coding agents.

## Execution rules

- One issue per PR unless explicitly stated otherwise.
- Later stages may not begin until the required predecessor contracts are merged.
- Each PR must include tests, documentation, rollback, and a 10th-Man countercase.
- Architecture is promoted only by evidence from the previous stage.

## Stages

1. Foundation hardening and end-to-end continuity.
2. Versioned capability contracts and deterministic registry.
3. Mission engine with authority envelopes and recovery.
4. Epistemic ledger and evidence lifecycle.
5. 10th-Man immune cascade and dormant wake protocols.
6. Sleep consolidation with quarantine and promotion boundaries.
7. OpenAI-compatible runtime integration and bounded context assembly.
8. Classical-ML divergence detectors and evaluation harness.
9. Inspectable skills and adapter-readiness pipeline.
10. Tauri HUD only after the kernel demonstrates stable utility.

## Definition of preliminary expansion complete

The kernel can:

- resume a session from durable state;
- create and recover bounded missions;
- register and invoke typed deterministic capabilities;
- preserve evidence and proposition state separately;
- detect and investigate divergence through the immune cascade;
- consolidate a session without contaminating belief or adaptation;
- invoke a local OpenAI-compatible model endpoint;
- run tests on Windows without Docker;
- explain every durable state transition through provenance.
